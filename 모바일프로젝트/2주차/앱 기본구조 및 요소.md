## 안드로이드 프로젝트 구조 Empty Activity 프로젝트 예 
```
1. 프로젝트 구성
프로젝트는 모듈로 구성되어 있으며, 여러 개의 모듈을 가질 수 있음

2. 모듈의 기본 파일 구조
각 모듈은 기본적으로 manifests, java, res 폴더 포함
각 모듈에 포함되는 파일들은 프로젝트를 만들 때 액티비티 유형에 따라 달라짐

3. Gradle Scripts
빌드를 위한 파일들로 구성
gradle은 안드로이드 앱의 빌드 도구
build.gradle: 빌드를 위한 다양한 설정 내용 저장

4. gradle 파일
프로젝트 수준의 build.gradle와 모듈 수준의 build.gradle 파일 존재 
```
## 모듈의 개념
```
소프트웨어 프로젝트의 독립적인 부분으로, 다른 모듈과 함께 작동하지만 독립적으로 개발 및 테스트 가능
각 모듈은 특정한 기능을 가지고 있으며, 다른 모듈에 비해 재사용성 높음

여러 개의 모듈을 가질 수 있다는 것 >>
안드로이드 프로젝트 내에 여러 개의 모듈 생성
한 모듈은 메인 앱의 ui 담당, 다른 모듈은 네트워킹 또는 DB 관련 기능 담당
```

## 안드로이드 어플리케이션의 주요 구성 요소
```
1. Source Code 소스 코드
주로 activity 클래스 포함, 애플리케이션의 비즈니스 로직과 사용자 인터페이스 구현
소스 코드는 앱의 동작과 사용자와 상호작용을 제어하는 역할

2. Resources 리소스
레이아웃, 이미지, 문자열 등 앱에서 사용되는 다양한 자원
리소스는 XML 파일 형태로 저장, 앱의 UI 및 기타 기능에 사용

3. Manifest
애플리케이션에 대한 중요한 메타데이터 정의
앱의 구성 요소, 권한 및 기타 설정, 운영체제에 의해 앱이 올바르게 실행되도록 도움

4. R 클래스
android 리소스의 ID 참조 클래스
앱의 리소스 ex) 이미지, 레이아웃 접근할 때 사용, 컴파일 시 자동 생성 
````

## 안드로이드  구조
### 1. 액티비티 Activity
```
- 사용자와 상호작용할 수 있는 화면 제공
- 화면 구성 요소를 조작하고 관리하는 역할
- JAVA 클래스로 정의, 확장자 .java 
```
### 2. 레이아웃 Layout
```
- 화면 구성 요소를 화면에 배치하고 구성하는 방법
- XML 형식으로 UI 설정, 확장자 .xml
- UI 요소의 배치와 디자인 정의
```
### 3. 레이아웃 인플레이션 Layout Inflation
```
XML 레이아웃에 정의된 화면 구성 내용을 읽어 화면 구성 요소를 생성해 화면 구성
주요 메소드 >>
- setContentView(): Activity의 UI 설정 메소드, 지정된 XML 레이아웃 화면에 표시
- findViewByID(): XML 레이아웃에 정의된 특정 뷰 참조 
```
## MainActivity 파일
```
- onCreate 메소드는 액티비티가 생성될 때 호출, 초기화 작업 수행
- setContentView(R.layout.activity.activity_main) 메서드 사용하여
activity_main.xml 파일을 액티비티의 사용자 인터페이스로 설정
```
## XML 레이아웃
```
- 레이아웃 파일 정의, ConstraintLayout이 사용
- layout_width와 layout_height는 뷰의 크기 지정: match_parent와 wrap_content 사용
- android:id 뷰의 고유 ID 설정해 코드에서 참고
app:layout_constraint...: 뷰의 위치와 관계 정의하는 속성
```
## 매니페스트 파일
```
안드로이드 시스템이 앱의 코드를 실행하기 전에 확보해야 하는 앱의 필수 정보를 시스템에 제공
```
## 리소스 파일 Resources
```
1. 리소스의 역할
코드에서 정적인 값 분리하여 관리 -> 유지보수와 관리 측면에서 유리
리소스를 사용함으로써 문자열, 색상, 크기, 레이아웃, 이미지 등 효율적 관리 가능

2. 리소스 파일 형식
대부분의 리소스는 XML파일 형식으로 작성
ex 레이아웃 파일은 UI 정의, 문자열 리소스는 앱에서 사용할 텍스트 관리

3. 프로젝트 내 리소스 저장 위치
모든 리소스는 프로젝트의 res 폴더 내에 저장, 이 폴더 안에는 다양한 리소스를 구별하기 위한 하위 폴더 존재

4. 리소스 이름 규칙
리소스 이름은 자바 변수 명명 규칙 다르며, 대문자나 공백 X
하위 폴더의 이름도 지정된 규칙 따르고, 잘못된 경우 오류 발생 
```
## R.java 파일
```
1. R.java의 역할
R.java 파일은 애플리케이션 내에서 사용되는 리소스의 ID를 저장하는 클래스
리소스의 유형별로 내부 클래스가 정의, 각 리소드드이 ID를 정수형 상수로 접근
=> 앱 코드에서 리소스에 쉽게 접근이 가능해짐

2. 리소스 사용방법
- java 코드에서의 참조 방법: R.리소스유형.리소스이름 형식으로 사용
- XML 파일에서의 참조 방법: @리소스유형/리소스이름 형식으로 사용
```
